# グルメサークル アプリ設計書

## 思想

**核心**: 「人」が中心。グルメはその人を知るための一つの切り口。
**コンセプト**: 信頼できる人の紹介だけでお店を知れる世界
**ポジション**: LinkedIn × グルメ（投稿で盛り上がるSNSではなく、人を知るためのコミュニティ）

### 禁止事項
- 星評価・スコア評価
- 匿名レビュー
- 広告課金による表示順操作

### 重視すること
- 紹介制・実名制
- 人間関係の可視化（紹介ツリー）
- ストーリー重視（なぜ薦めるかが見える）
- コンテキスト（シーンに合わせたお店選び）

---

## ルート構成

```
ディレクトリ: app/app/ （ヘッダー・フッターなし、サイドバーレイアウト）
joinの先 → /app/dashboard がログイン後のホーム

/app/dashboard          → メインダッシュボード（MVP）
/app/profile/[id]       → メンバープロフィール（将来）
/app/post               → ストーリー投稿（将来）
/app/discover           → 発見・検索（将来）
/app/tree               → 紹介ツリー可視化（将来）
/app/settings           → 設定（将来）
```

## メンバーカード設計

```
┌──────────────────────┐
│  [アバター写真]        │
│                      │
│  田中 一郎            │  ← 実名
│  株式会社〇〇 代表取締役│  ← 役職・会社
│                      │
│  🛡️ 信頼 85  📝 12件  │  ← 信頼スコア・おすすめ数
│                      │
│  [接待向き] [和食]     │  ← よく使うタグ（2-3個）
│                      │
│  紹介者: 佐藤さん      │  ← 誰の紹介か
└──────────────────────┘
```

### カードの情報要素
- プロフィール写真
- 氏名（実名）
- 役職・会社名
- 信頼スコア（紹介実績ベース）
- おすすめ投稿数
- よく使うコンテキストタグ（2-3個）
- 紹介者名（誰経由で参加したか）

## デザインルール

### カラー
- **サイドバー**: bg-gray-900（ダーク）
- **メインコンテンツ**: bg-gray-50（ライトグレー）
- **カード**: bg-white, rounded-2xl, border, shadow-sm
- **アクセント**: amber系（グルメLPと統一）
- **信頼スコア**: amber badge
- **タグ**: bg-amber-50 text-amber-700 border-amber-200

### タイポグラフィ
- 見出し: Noto Serif JP（和の上品さ）
- 本文: システムフォント
- 数値: tabular-nums

### コンポーネント
- カード: hover:shadow-md, transition
- ボタン: rounded-xl
- タグpill: rounded-full
- アバター: rounded-full, border-2 border-white shadow

## モックデータ

### メンバー（8-12人）
- 既存の members ページのデータを活用
- 追加: 信頼スコア、おすすめ数、コンテキストタグ、紹介者

### 業種カテゴリ
- 全員 / 経営者 / IT・テック / 飲食 / 不動産 / クリエイティブ / コンサル

### コンテキストタグ
- デート向き / 接待・会食向き / 一人で集中 / 家族向き / 記念日に / カジュアル

## 将来の拡張（MVP後）

### Phase 2: プロフィール & お店詳細
- /app/profile/[id] — メンバー詳細（紹介ツリー、投稿一覧）
- /app/restaurant/[id] — お店詳細（ストーリー一覧）

### Phase 3: 投稿 & 発見
- /app/post — ストーリー型投稿（写真+エピソード+タグ）
- /app/discover — タグ/エリア/AI検索

### Phase 4: AI & 高度な機能
- AIレコメンド（好みマッチング）
- 紹介ツリー可視化（D3.js or similar）
- ストーリー自動整形
- 写真の雰囲気解析

### Phase 5: マネタイズ
- 経営者プレミアム（接待用レコメンド、個室検索、AIコンシェルジュ）
- コミュニティ月額課金との統合

## 技術スタック（現状）

- **フレームワーク**: Next.js（App Router）
- **スタイリング**: Tailwind CSS
- **アニメーション**: GSAP
- **アイコン**: Lucide React
- **データ**: モック（ハードコード） → 将来 Supabase
- **認証**: なし（モック） → 将来 Supabase Auth
